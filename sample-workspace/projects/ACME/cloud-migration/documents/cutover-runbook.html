Cutover runbook — Acme cloud migration

A worked example document, shown inline in the project portal's Documents tab. Any PDF, HTML, PNG or Markdown file placed in a project's documents/ folder is rendered inline; Office files (docx/xlsx) are shown inline once rendered to PDF (see scripts/render-doc-pdfs.ps1), otherwise offered as a download.

1. Pre-cutover

  • Confirm backup of the on-prem datastore is complete and restorable.
  • Freeze schema changes 24h before the window.
  • Notify Acme stakeholders of the maintenance window.

2. Cutover

  • Switch DNS to the new cloud endpoint.
  • Run the data-sync to catch the final delta.
  • Smoke-test the critical user journeys.

3. Rollback trigger

If smoke tests fail or error rate exceeds the agreed threshold, revert DNS and re-open the on-prem endpoint. The decision is the cutover lead's; document it on the relevant decision-card.
